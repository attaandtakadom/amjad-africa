📚 الإجراءات الفرعية (Sub) والدوال (Functions)

فيجوال بيسك دوت نت | شرح كامل مع Immediate Window ونقاط التوقف (Breakpoints)

✨ الفرق الأساسي في الاستدعاء

📌 الفرق الأهم في طريقة الاستدعاء:
  • الإجراء الفرعي (Sub): يُستدعى باستخدام Call أو مباشرة باسمه
  • الدالة (Function): تُستدعى باستخدام جملة الإسناد (Assignment Statement) أو داخل تعبير
' مثال 1: استدعاء الإجراء الفرعي Sub
Call GreetUser("أحمد") ' باستخدام Call
GreetUser("أحمد") ' مباشرة بدون Call

' مثال 2: استدعاء الدالة Function - باستخدام الإسناد
Dim result As String = addMessage(45) ' تخزين النتيجة في متغير
Print addMessage(45) ' طباعة النتيجة مباشرة
MsgBox(addMessage(45)) ' استخدام الدالة داخل دالة أخرى

🔹 استدعاء الإجراء الفرعي (Sub)

Private Sub ConstructMessage(V As Integer)
    MsgBox "Value of V is = " & V
End Sub

' طرق الاستدعاء:
Call ConstructMessage(45)
ConstructMessage(45)

⚠️ لا يمكن استخدام Sub في جملة الإسناد لأنه لا يعيد قيمة

🔹 استدعاء الدالة (Function)

Private Function addMessage(Val As Integer) As String
    addMessage = "My Age is " & Val
End Function

' طرق الاستدعاء:
Dim m1 As String = addMessage(45) ' إسناد إلى متغير
Print addMessage(45) ' طباعة مباشرة
MsgBox(addMessage(45)) ' داخل دالة أخرى

✅ يمكن استخدام Function في أي تعبير لأنها تعيد قيمة

📖 شرح تفصيلي مع أمثلة عملية

🔸 المثال الكامل :

' تعريف الدالة
Private Function addMessage(Val As Integer) As String
    addMessage = "My Age is " & Val
End Function

' استدعاء الدالة في زر الأمر
Private Sub Command1_Click()
    Dim m As String
    m = addMessage(45) ' جملة الإسناد - استدعاء الدالة
    Print m ' طباعة النتيجة: My Age is 45
End Sub
💡 توضيح مهم: عند كتابة m = addMessage(45)، يتم تنفيذ الدالة وإرجاع القيمة "My Age is 45" التي تُخزن في المتغير m.

🔸 مقارنة شاملة بين Sub و Function

المقارنة Sub (إجراء فرعي) Function (دالة)
طريقة الاستدعاء Call Name() أو Name() Variable = Name() أو Print Name()
إرجاع قيمة ❌ لا يعيد قيمة ✅ يعيد قيمة واحدة
استخدام في التعبيرات ❌ لا يمكن ✅ يمكن استخدامها في العمليات
كلمة الإنهاء End Sub End Function

🐞 النافذة السريعة (Immediate Window) ونقاط التوقف (Breakpoints)

📌 ما هي النافذة السريعة (Immediate Window)؟
هي نافذة تستخدم لمعرفة قيم المتغيرات، تنفيذ بعض الأوامر، وتغيير قيم الخصائص أثناء تنفيذ البرنامج.

🔹 خطوات استخدام النافذة السريعة:

1. أضف نقطة توقف (Toggle Breakpoint) من قائمة Debug أو بالضغط F9
2. شغل البرنامج بالضغط على زر الأمر (Command Button)
3. عندما يتوقف التنفيذ عند نقطة التوقف، افتح النافذة السريعة:
    - من قائمة View → Immediate Window
    - أو اختصار Ctrl + G
4. اكتب الأمر ? متغير أو Print متغير لمعرفة قيمته
5. يمكنك أيضاً تغيير قيمة المتغير أثناء التنفيذ

💻 أمثلة على استخدام النافذة السريعة (Immediate Window):

' معرفة قيمة المتغير m
? m
Print m
' النتيجة: My Age is 45

' تنفيذ دالة مباشرة
? addMessage(30)
' النتيجة: My Age is 30

' تغيير قيمة خاصية في النموذج
Form1.Caption = "قيد التصحيح"
Text1.Text = "اختبار"

' تنفيذ إجراء فرعي
Call GreetUser("أحمد")

🔹 مثال عملي كامل:

' الكود الموجود في النموذج Form1
Private Sub Command1_Click()
    Dim m As String
    m = addMessage(45) ' ← ضع نقطة توقف هنا (F9)
    Print m
End Sub

Private Function addMessage(Val As Integer) As String
    addMessage = "My Age is " & Val
End Function

' عند تشغيل البرنامج وتوقف التنفيذ:
' 1. افتح Immediate Window (Ctrl+G)
' 2. اكتب: ? m (ستظهر قيمة m وهي Nothing حتى الآن)
' 3. اضغط F8 للتنفيذ خطوة بخطوة
' 4. اكتب مرة أخرى: ? m (ستظهر "My Age is 45")
🎯 اختصارات مهمة في وضع التصحيح (Debug Mode):
  • F9 - إضافة/إزالة نقطة توقف (Toggle Breakpoint)
  • F5 - تشغيل البرنامج / متابعة التنفيذ
  • F8 - تنفيذ سطر بسطر (Step Into)
  • Shift + F8 - تنفيذ بدون الدخول إلى الدوال (Step Over)
  • Ctrl + G - فتح النافذة السريعة (Immediate Window)
  • Ctrl + Shift + F8 - الخروج من الدالة الحالية (Step Out)
